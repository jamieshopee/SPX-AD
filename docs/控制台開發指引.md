# 控制台開發指引

Version: v0.3.1  
Last Updated: 2026-07-05  
Scope: 控制台、Canvas、Editor、Template、Style、Project State 與共用模組的開發規則。

## What's New

- 新增三商品商品身份識別規則：`id → filename → position`。
- 新增 Asset shared modules：分類、trim、Canvas payload 產生。
- 新增 State Boundary / Render Boundary 規則，明確定義 Main Canvas、Thumbnail、Batch Render、Project State 與 Asset Payload 的讀寫責任。
- `layoutStates` 以 `placementId|templateId` 保存，避免不同尺寸互相污染。
- Thumbnail / Batch Render 只能讀 layout state，不得回寫 transform。

## 整體架構

控制台管理 UI 與工作區資料；Canvas 只負責渲染與 capture；Editor 與控制台共用 runtime；Template 保存排版結構；Style 保存視覺樣式；Thumbnail Render 使用 hidden iframe；Project State 負責匯入、匯出與恢復完整工作區。

```text
index.html
  ↓
src/app.js
  ↓
Project State
  ↓
canvas.html
  ↓
template.json + styles/{styleId}.json
  ↓
Render / Thumbnail / Download
```

## 共用模組

- `js/product-slot-utils.js`：商品角色判斷。
- `js/box-transform-utils.js`：拖曳、縮放、旋轉、wheel 縮放、reset transform。
- `js/template-loader.js`：Template + Style 載入與合併。
- `js/bn-image-utils.js`：影子、baselineRatio 與影像共用處理。
- `js/asset-classifier.js`：Logo 排序、三商品分類、1人＋1品分類。
- `js/asset-processing.js`：素材 trim / autoTrim。
- `js/asset-render-payload.js`：將素材轉成 Canvas postMessage payload。
- Project State Serializer：`exportSingleState()`、`exportProjectState()`、`importState()`。
- Thumbnail Queue：quick thumbnail 與 hidden iframe full thumbnail。
- Layout State：Canvas 回傳，job 保存，下載與暫存共用。
- Accordion：右側素材區收合狀態依素材模式初始化。

## 事件流程

```text
CSV / 暫存 JSON
  ↓
控制台建立或恢復 Project State
  ↓
canvas.html?template=.../template.json&style=XX
  ↓
Canvas Render
  ↓
Layout State
  ↓
Thumbnail
  ↓
Download PNG / Project State JSON
```

## CSV 與暫存

CSV = 新專案。

匯入 CSV 前必須清空目前 Project State：

- jobs
- currentJob
- layoutState
- transform state
- Logo / Product / Person / SingleProduct state
- thumbnails
- selected style
- imported temp state
- local workspace cache
- pending thumbnail queue

匯入暫存 JSON = 恢復舊專案。

只有匯入暫存 JSON 才可以恢復文字修改、素材、Style、transform、layoutState 與 thumbnail。CSV 匯入不得 merge 舊狀態。

## Template / Style

Template 固定為：

```text
templates/{size}/template.json
```

Template 只控制排版結構、座標、尺寸、圖層、Logo、商品、Person、SingleProduct、Transform 初始排版等。

Style 固定為：

```text
templates/{size}/styles/{styleId}.json
```

Style 只控制：

- `background`
- `infoGraphic`
- `headlineColor`
- `subHeadlineColor`
- `smallTextColor`

背景與資訊圖來源：

```text
assets/source/{size}/backgrounds/bg_XX.png
assets/source/{size}/info/info_XX.png
```

`scripts/generate-style-json.js` 是 Developer Tool，可用來產生 / 補齊 `styles/*.json`。Runtime 不得依賴此工具。

## Style 下拉

Style 下拉應讀取目前 job 的 `style`。切換 Style 時：

- 更新 job.style。
- 送出 `bn-style-apply`。
- Canvas 更新背景、資訊圖與文字顏色。
- 不清除文字、Logo、商品、1人＋1品、transform 或 layoutState。
- 更新目前工單 thumbnail。
- Project State 匯出需保存目前 style。

## 匯入素材資料夾

素材資料夾只提供原始圖片來源。CSV 匯入時可保留已選素材資料夾，但不能保留前一輪 jobs 或 transform。

帶入規則：

- Logo 依 `LOGO_01 / LOGO_02 / LOGO_03`。
- 三商品依 `product-slot-utils.js` 判斷 position。
- 1人＋1品依 `_人`、`_品` 判斷。
- 三商品與 1人＋1品互鎖。

## Transform

三商品 Product 與 1人＋1品 SingleProduct 共用 `box-transform-utils.js`。可用能力：

- 拖曳
- 縮放
- 旋轉
- wheel 縮放
- transform 狀態保存
- transform 狀態還原
- 恢復預設位置

恢復預設位置必須先清除 user transform state，再重新套用 Template 初始 layout，最後更新 DOM、layoutState、job state 與 thumbnail。

## Product Slot

商品角色判斷只由 `BNProductSlotUtils.detectProductPosition(filename)` 負責。

優先語意命名：

- `_主品` → position 0
- `_左配品` → position 1
- `_右配品` → position 2

相容舊命名：

- `_01 / _1` → position 0
- `_02 / _2` → position 1
- `_03 / _3` → position 2

Position 定義固定：0 主品、1 左配品、2 右配品。不要把 position 當成畫面左中右。

## 商品身份識別（Product Identity）

所有三商品相關流程：

- Main Canvas Restore
- Batch Render
- Project State Restore
- Thumbnail Restore

必須依照：

```text
id
↓
filename
↓
position（最後 fallback）
```

禁止：

- 只依 array index 對應商品
- 只依 position 對應商品

原因：

position 代表目前排列位置，不代表商品身份。

## Thumbnail

CSV 匯入後立即建立 quickThumbnail；正式 thumbnail 由 queue 逐張生成。Hidden iframe 必須使用指定 job 的 `template.json` 與 `styleId`，不能依賴 currentJob。

Capture 流程：

```text
hidden iframe ready
  ↓
post text / logos / products / person / singleProduct / layoutState
  ↓
bn-capture
  ↓
bn-snapshot
  ↓
job.thumbnail
  ↓
update left panel DOM
```

Thumbnail queue 一次只處理少量工作，避免阻塞主畫面。使用者拖曳 transform 時應 debounce 更新目前工單 thumbnail。

## Project State

Project State JSON 保存：

- CSV / jobs
- 文字
- size / template / style
- Logo / Product / Person 素材 dataUrl
- `layoutStates`：依 `placementId|templateId` 保存每個尺寸 / template 的 transform 狀態
- `layoutState`：legacy 相容欄位，不作為跨尺寸 fallback 的主要依據
- thumbnail

素材存在 JSON assets 中，jobs 以 asset id 引用，避免同一素材重複存多份。

## State Boundary / Render Boundary

State Boundary 的核心規則：只有 Main Canvas 的 active edit flow 與 Project State Import 可以寫入 `job.layoutStates`。Thumbnail、Batch Render、Asset Payload 與未來 Photoshop Pipeline 都不得回寫 layout state。

讀寫邊界：

| 模組 / 流程 | 可讀取 | 可寫入 | 不可寫入 |
|---|---|---|---|
| Main Canvas | active job、template、style、assets、目前 key 的 `layoutStates` | active job 的 `layoutStates[placementId|templateId]` | 非 active job、其他尺寸 key |
| Thumbnail hidden iframe | 指定 job 的文字、素材、style、目前 key 的 `layoutStates` | `job.thumbnail`、`job.thumbnailStatus` | `layoutState`、`layoutStates`、active placement/template |
| Batch Render | render job 自己的文字、素材、style、`layoutStates` | PNG ZIP、可更新 UI thumbnail | transform、`layoutStates`、currentJob 狀態 |
| Project State Export | jobs、assets、style、`layoutStates`、thumbnail | JSON 檔案 | runtime transform |
| Project State Import | 暫存 JSON | jobs、assets、`layoutStates`、thumbnail | 不可 merge 舊工作區殘留狀態 |
| Asset Payload | filenames、asset handles、template default layout | `bn-logos`、`bn-product-add`、`bn-person-add`、`bn-single-product-add` | Project State、thumbnail、layoutState |
| Photoshop Pipeline（未來） | 原始素材、處理狀態、approved assets | processed / approved asset mapping | Canvas DOM、Render Engine、`layoutStates` |

Render restore 順序固定：

```text
load template
  ↓
apply text
  ↓
apply asset payload
  ↓
wait images
  ↓
apply layoutStates[current placement|template]
  ↓
capture / export
```

`layoutStates` 規則：

- key 固定為 `placementId|templateId`。
- 有 `layoutStates` map 時，只能讀目前 key。
- 如果目前 key 沒有 state，使用該尺寸 / template 的預設 layout。
- 不得 fallback 到其他尺寸的 layoutState。
- 三商品未來可另行設計 Linked Product Transform，但 Phase 1 不做跨尺寸同步。
- 1人＋1品不做跨尺寸同步；Person 與 SingleProduct 依各尺寸 template 預設或該尺寸自己的 layoutState。

Thumbnail 與 Batch Render 規則：

- Hidden iframe 只能讀指定 job，不得依賴 currentJob。
- Hidden iframe 產縮圖不得呼叫 `setJobLayoutState()`。
- Batch Render 可讀 layoutState 並套到 render iframe，但不得回寫 transform。
- 若背景 render 使用主 iframe，必須用 load token 防止舊 callback 寫到新 iframe。

Asset Payload 規則：

- Payload 只負責素材初始套入 Canvas。
- Payload 不負責 restore 使用者 transform。
- Restore 一律在素材 payload 套完後執行。
- Photoshop Pipeline 未來只輸出 processed / approved assets，不直接控制 Render Engine。

## 下載與匯入暫存

- 下載單張暫存：輸出 `single-state.json`。
- 批次產圖：輸出 PNG ZIP，內含 `project-state.json`。
- 匯入暫存：接受 `single-state.json` 或 `project-state.json`。

匯入暫存後左側縮圖應立即顯示，不需要等待重新 capture。

## 重設工作區

重設工作區需先顯示確認 Dialog。確認後清除：

- jobs / currentJob
- CSV state
- Logo / Product / Person / SingleProduct
- Template / Style 狀態
- transform / layoutState
- thumbnails
- Project State / imported temp state
- localStorage / sessionStorage / IndexedDB
- pending thumbnail queue

重設後 UI 回到初始狀態：不顯示已套用文字、不顯示素材 Badge、不顯示 CSV Badge、模板與樣式顯示初始 placeholder。

## 重要規則

- 不重新實作商品角色判斷。
- 不重新實作 Transform Controller。
- 不讓控制台與 Editor 維護不同邏輯。
- 不在多個 JS 寫相同規則。
- 新增共用能力優先抽 shared module。
- Style 不得覆寫 layout、座標、尺寸或 transform。
- CSV 匯入不得讀取舊工作區狀態。
- 暫存匯入不得依賴原本素材資料夾。
